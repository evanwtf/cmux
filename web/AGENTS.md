# web — agent guide

Component-specific rules for `web/`. Cross-cutting repo rules: [`/AGENTS.md`](../AGENTS.md). Deeper detail: [`web/README.md`](README.md) and the `cmux-backend` skill.

## Overview
Next.js app deployed as the Vercel `manaflow/cmux` project: the marketing site, `cmux.com/docs`, Stack Auth handlers, feedback endpoint, Stripe, device push, and the **Cloud VM control plane** (`app/api/vm`). Data in Postgres via Drizzle.

## Tech Stack
TypeScript, Next.js, bun, Drizzle ORM (`drizzle-kit`), PostgreSQL (local Postgres in Docker), ESLint, Stack Auth, Stripe.

## Environment & Dependencies
- `bun` and Docker (local Postgres).
- Secrets live in `~/.secrets/`, not in-repo: provider secrets in `~/.secrets/cmux.env`, Stack/web secrets in `~/.secrets/cmuxterm-dev.env`. `web/.env.local` is **not** used for local dev.
- `bun dev` sources those secrets, starts this worktree's Docker Postgres, applies Drizzle migrations, then starts Next.js on `CMUX_PORT` → `PORT` → `3777`.

## Commands
```bash
bun install
bun dev                     # Postgres + migrations + Next.js
bun test                    # bun test
bun run test:db:behavior    # DB behavior tests
bun tsc --noEmit            # typecheck (matches the web-typecheck CI job)
bun run lint                # eslint
bun run db:generate         # generate a Drizzle migration from schema.ts
bun run db:migrate          # apply migrations
bun run db:check            # validate migration state
bun run db:up | db:down | db:reset | db:status   # local Postgres lifecycle
```

## Project Layout
- `app/` — routes; `app/[locale]` (site + docs), `app/api/{vm,stripe,devices,device-tokens,notifications,feedback,analytics,github-stars}`.
- `db/` — `schema.ts`, `client.ts`, `config.ts`, generated `migrations/`.
- `messages/` — i18n catalogs (`en.json`, `ja.json`, …); `i18n/`, `instrumentation.ts`, `drizzle.config.ts`, `docker-compose.db.yml`.

## Guardrails
- **Always** access the DB through Drizzle; localize user-facing strings in **every** `web/messages/*.json` (see `cmux-localization`).
- **Never** commit secrets or rely on `web/.env.local`.
- **Use caution:** `db/migrations/` is generated by `drizzle-kit` — change `db/schema.ts` then `bun run db:generate`, don't hand-edit migrations. Lockfile is `bun.lock`.
